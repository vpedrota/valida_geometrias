core_functions.py

O schema foi definido de acordo com as normas do GeoJSON e aceitam todas as geometrias oficialmente suportadas. Consulte a seguir o schema definido:

 schema = {
     "type": "object",
     "properties": {
         "type": {
                     "enum" : ["Polygon", "MultiPolygon", "LineString", "Point", "MultiPoint", "MultiLineString", "GeometryCollection"]
                 },
         "coodinates": { "type": "array"}
     },
     "required": ["type", "coordinates"]
 }

Nesta parte apenas a verificação do formato é feita, o conteúdo não é verificado. A verificação do conteúdo é feita posteriormente utilizando o Postgis.
core_functions.json_schema_checker(request: dict) → dict

Esta função realiza a validação do json de entrada com o schema definido, se houver erros, retorna um json com os erros. Se não houver erros, retorna None. O schema foi definido de acordo com as normas do GeoJSON e aceitam todas as geometrias oficialmente suportadas.

Parâmetros

request (dict) – este argumento é um dicionário que representa o json de entrada. Nele deve ser definido o campo “type” e o campo “coordinates”.

Retorna

retorna um dicionário com os erros encontrados. Se não houver erros, retorna None.

Tipo de retorno

dict

Exemplos:
>>> json_schema_checker({'type': 'Polygon', 'coordinates': [[[1, 2], [3, 4], [5, 6], [1, 2]]]})
None

>>> json_schema_checker({'type': 'Polygon', 'coordinotes': [[[1, 2], [3, 4], [5, 6], [1, 2]]]})
